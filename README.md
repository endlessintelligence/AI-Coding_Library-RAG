## 一、需求来源与设计背景

### 1.1 占座霸座问题

**痛点来源：小组成员的亲身经历与长期观察。**

每逢期末考试季，图书馆各楼层自习区出现大量"书在人空"的占座现象——座位上仅放置几本个人书籍或水杯，却长时间无人使用。后来者即使找到空位也不敢使用，造成座位资源严重浪费。图书馆工作人员巡查频次有限，无法及时发现和处理。这一现象在 2-5F 自习区尤为突出，-1F 电脑区也未能幸免。

**设计目标：** 建立"预约 → 签到 → 暂离 → 签退"的完整闭环，通过 15 分钟签到窗口、10 分钟暂离限制、每学期 3 次违规暂停预约资格等机制，从制度层面杜绝占座行为。

### 1.2 图书馆资源使用不便

**痛点来源：小组成员的实际使用体验与问卷调查反馈。**

多数读者仅将图书馆视为"自习场所"，对图书馆提供的丰富服务知之甚少：

- 不清楚图书分类（A-F / G-L / M-R / S-Z）与楼层的对应关系，找书耗费大量时间
- 不了解馆内服务：文献传递与馆际互借、查收查引、学科服务、信息素养教育、空间预约等
- 遇到证件遗失、借阅异常、违规处罚等问题时，不知道如何联系工作人员、去哪里寻找帮助

**设计目标：** 基于 RAG 技术构建智能问答助手，将图书馆知识库（官网公开信息、义馆工作手册、排班规则等）向量化存储，用户自然语言提问即可获得精准回答，并可在必要时引导至对应楼层的工作人员。

> **注：** 截至项目完成，校图书馆官网尚未提供基于 RAG 技术的智能问答服务，本系统填补了这一空白。

### 1.3 扩展性设计

系统采用 **三轴架构（Rules / Resources / Personnel）**，为未来扩展至完整图书馆管理系统预留了清晰的扩展路径：

| 当前覆盖 | 未来可扩展 |
|---------|-----------|
| 座位预约规则 | 图书借还规则、研讨室预约规则、设备借用规则 |
| 楼层分布与图书分类 | 可借用设备信息、研讨室配置、座位热力图 |
| 管理员值班信息 | 志愿者排班签到、活动人员安排、学科馆员对接 |

## 二、系统概述

### 2.1 系统名称

**图书馆座位预约及智能问答系统**

### 2.2 核心解决的两大问题

| 问题 | 解决方案 | 核心技术 |
|------|---------|---------|
| 占座霸座、座位资源浪费 | 座位预约闭环 + 违规处罚机制 | FastAPI + SQLAlchemy + WebSocket |
| 图书馆资源使用不便、不知如何索引和使用 | 智能问答助手 + 工作人员引导 | RAG + LangGraph + Qwen-Max |

### 2.3 系统特性

- **座位预约闭环**：预约 → 签到（15分钟窗口）→ 暂离（每小时1次，每次10分钟）→ 签退
- **违规处罚**：超时未签到 / 暂离超时未返 / 巡查发现无人 → 计违规 → 每学期 3 次暂停预约资格 180 天
- **管理员功能**：释放已签到但占用的座位 + 上传含时间戳的证据图片 + 用户封禁/解封管理
- **多智能体问答**：Master 意图路由 → 四轴 Agent 分工处理
- **RAG 知识库**：图书馆官网信息、义馆工作手册、排班规则等 → 分块 → 向量化 → 语义检索
- **实时推送**：WebSocket 广播座位状态变更，SSE 流式输出 AI 回答
- **值班人员导航**：根据当前时间自动匹配值班人员所在楼层，引导用户前往对应办公台

### 3.2 技术栈

| 层级 | 技术 | 用途 |
|------|------|------|
| **后端框架** | Python 3.9 + FastAPI | RESTful API + SSE + 静态服务 |
| **数据库 ORM** | SQLAlchemy 2.0 | 数据模型与 CRUD |
| **数据库** | SQLite | 单文件零配置，存储用户/座位/预约/违规等 |
| **向量数据库** | ChromaDB | RAG 知识库持久化存储 |
| **LLM** | Qwen-Max (阿里云百炼) | 意图分类 + 领域回答 |
| **嵌入模型** | Text-Embedding-V2 (阿里云百炼) | 文档/问题向量化 |
| **多智能体框架** | LangGraph (StateGraph + MemorySaver) | Agent 编排状态图 |
| **文本分块** | LangChain RecursiveCharacterTextSplitter | 文档智能分块 |
| **前端** | 原生 HTML + CSS + JavaScript (SPA) | 无框架依赖，即开即用 |
| **实时通信** | WebSocket + SSE | 座位状态广播 + 流式 AI 回答 |
| **定时任务** | APScheduler | 30 秒扫描超时预约/暂离 |
| **JWT** | 手动 HMAC-SHA256 | 无额外依赖的令牌认证 |
| **AI Coding** | opencode | 项目全程 AI 辅助开发 |

### 3.3 大模型技术应用

| 技术点 | 模型 | 参数 | 应用位置 |
|--------|------|------|---------|
| **意图分类** | Qwen-Max | temperature=0.1 | Master Agent 路由判断 |
| **领域问答** | Qwen-Max | temperature=0.3 | Rules/Resources/Personnel/FAQ Agent |
| **文本嵌入** | Text-Embedding-V2 | dim=1536 | RAG 文档与查询向量化 |
| **语义检索** | ChromaDB 余弦相似度 | top_k=5 | 知识库片段召回 |
| **上下文注入** | System Prompt + RAG 片段 | — | FAQ Agent 动态拼装 |

### 3.4 AI Coding 辅助开发

本项目全程使用 opencode 作为 AI Coding 助手，覆盖以下环节：

- 项目架构设计与目录结构搭建

- RAG 流水线（分块 → 嵌入 → 存储 → 检索）实现

## 四、多智能体系统

### 4.1 架构设计：三轴 + 路由 + 兜底

```
用户提问 → Master Agent ─┬→ Rules Agent (规则)
                         ├→ Resources Agent (资源)
                         ├→ Personnel Agent (人员)
                         └→ FAQ Agent + RAG (兜底)
```

### 4.2 各 Agent 职责

| Agent | 职责 | 知识来源 |
|-------|------|---------|
| **Master** | 意图识别与路由，输出 RULES/RESOURCES/PERSONNEL/FAQ | 系统 Prompt |
| **Rules** | 座位预约规则、签到/暂离/签退规则、违规处罚、证件管理 | 系统 Prompt |
| **Resources** | 楼层分布、图书分类、借还流程、馆内服务介绍 | 系统 Prompt |
| **Personnel** | 值班排班、办公台位置、人员引导 | 系统 Prompt + 排班表 |
| **FAQ** | 开放时间、文明守则、综合概况；超出系统能力时引导至工作人员 | RAG 知识库 + 系统 Prompt |

### 4.3 状态图 (StateGraph)

```
master → (rules / resources / personnel / faq) → summarize → END
```

### 4.4 值班排班逻辑

- 每天每个时间段 **仅 1 名值班人员** 负责全馆管理
- 值班人员固定在某个楼层的办公台（-1F 电脑区工作台 / 1F 总服务台 / 2F 借阅工作台 / 3F 阅览区工作台 / 4F 自习区工作台 / 5F 工作台）
- 每个问题自动注入当前时间（如 `[当前时间：周三 18:30]`），LLM 根据排班表匹配对应楼层
- 当回答需要"联系工作人员"时，必须附带当前时段的办公台位置

## 五、RAG 检索增强生成

### 5.1 数据来源

| 来源 | 内容 |
|------|------|
| 图书馆官网 | 读者文明守则、入馆指南、证件管理、违规处理、电子资源规定 |
| 义馆工作手册 | 工作地点、各班次工作事项、馆员须知、各楼层注意事项 |
| 中国图书馆分类法 | 图书分类排架规则、索书号排列方法 |

### 5.2 RAG 流水线

```
源文档 (docx) ─→ 分段提取 (章节) ─→ RecursiveCharacterTextSplitter
                                              ↓
             用户提问 ─→ text-embedding-v2 向量化 ←── ChromaDB 存储
                                              ↓
                 余弦相似度检索 top_5 ─→ 注入 Prompt ─→ Qwen-Max 生成
```

### 5.3 知识库统计

- 源文档段落：12 个章节
- 分块数量：45 个文本块
- 向量维度：1536 (text-embedding-v2)
- 存储引擎：ChromaDB (PersistentClient)

### 5.4 回答机制

- 系统 Prompt 包含基本概况 + 文明守则 + 值班信息 + 引导策略
- RAG 检索片段作为"参考知识库内容"动态注入
- LLM 基于 Prompt + 检索片段 + 自身知识生成回答
- 每条回答末尾标注："回答由模型生成，具体请以实际情况为准。"

## 六、座位预约闭环流程

### 6.1 完整状态流转

```
                     ┌──────────┐
                     │  预约成功  │ (pending)
                     └────┬─────┘
                          │ 15分钟内签到
                          ▼
                     ┌──────────┐
              ┌──────│  已签到   │──────┐
              │      │(checked_in)     │
              │      └────┬─────┘      │
              │ 暂离      │            │ 签退
              ▼           │            ▼
        ┌─────────┐      │      ┌──────────┐
        │  暂离中   │←────┘      │  已完成   │
        │temp_leave│  归位      │completed │
        └────┬────┘            └──────────┘
             │ 10分钟内未返回
             ▼
        ┌─────────────┐
        │  自动释放     │ (auto_released + 记违规)
        └─────────────┘
```

### 6.2 规则参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 签到窗口 | 15 分钟 | 预约开始后 15 分钟内必须签到 |
| 暂离时长 | 10 分钟 | 每次暂离不得超过 10 分钟 |
| 暂离频率 | 每小时 1 次 | 每小时最多暂离 1 次 |
| 违规上限 | 3 次/学期 | 累计 3 次暂停预约资格 180 天 |
| 单次预约 | 1 个座位 | 每人同时只能预约 1 个座位 |

### 6.3 违规触发场景

1. **超时未签到**：预约开始后超过 15 分钟未签到 → 自动释放 + 违规
2. **暂离超时未返**：暂离超过 10 分钟未归位 → 自动释放 + 违规
3. **管理员释放**：管理员巡查发现已签到但无人（暂离状态除外）→ 释放 + 违规 + 证据图片

## 七、系统功能模块

### 7.1 用户端

- **注册与登录**：学号注册，JWT 令牌认证
- **座位预约**：三步流程（楼层 → 日期+时段 → 座位图选座）
- **签到/暂离/签退**：预约卡片一键操作
- **取消预约**：预约开始前可自主取消
- **预约记录**：查看历史记录，可删除已结束的预约，点击有效预约跳转到对应楼层和时段
- **智能问答**：流式 SSE 聊天，实时显示 Agent 路由过程和思考动画
- **实时更新**：WebSocket 广播座位状态变更

### 7.2 管理员端

- **释放座位**：按座位号释放已签到但占用的座位 + 上传含时间戳证据图片
- **用户管理**：查看所有用户、暂停/解封账号（管理员不可被暂停）
- **释放记录**：查看历史释放记录（含证据标记）

### 7.3 实时通信

- **WebSocket**：座位预约/签到/暂离/释放等操作后广播状态变更
- **SSE (Server-Sent Events)**：AI 问答流式输出，实时显示 Agent 路由步骤

### 7.4 定时任务

APScheduler 每 30 秒执行：

1. **释放超时未签到的预约**：pending 状态 + 超过签到窗口 → auto_released + 违规
2. **释放超时未归的暂离**：temp_leave 状态 + 超过暂离时限 → auto_released + 违规
3. **违规处罚**：累计 3 次违规 → penalty_until = now + 180 days

## 十、部署与运维

### 10.1 环境要求

### 10.2 环境变量 (.env)

| 变量 | 说明 | 默认值 |
|------|------|--------|
| MODEL_API_KEY | 阿里云百炼 API Key | — |
| EMBEDDING_API_KEY | 嵌入 API Key（同 MODEL_API_KEY） | — |
| MODEL_NAME | LLM 模型名 | qwen-max |
| EMBEDDING_MODEL_NAME | 嵌入模型名 | text-embedding-v2 |
| JWT_SECRET | JWT 签名密钥 | library-seat-dev-key-2026 |
| CHECKIN_WINDOW_MINUTES | 签到窗口（分钟） | 15 |
| TEMP_LEAVE_MINUTES | 暂离时限（分钟） | 10 |
| MAX_RESERVATION_PER_PERSON | 每人同时预约数 | 1 |
| VIRTUALIZATION_CREDIT_MAX_VIOLATIONS | 违规上限 | 3 |

.venv\Scripts\python.exe -c "from rag.ingest import ingest; ingest(force_rebuild=True)"
